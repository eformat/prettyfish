Most useful Mermaid Sequence patterns

  • participant: Declare a participant in the sequence with optional alias
  • actor: Declare a human actor with person icon
  • solid arrow message: Send a synchronous message with solid arrowhead
  • solid dashed message: Send an asynchronous message with dashed solid arrow
  • open arrow message: Send a message with open arrowhead (no fill)
  • open dashed message: Send a dashed message with open arrowhead
  • cross message: Send a message with cross/X mark on target
  • cross dashed message: Send a dashed message with cross on target

Useful Mermaid Sequence syntax

Element Syntax What it does
participant participant Alias as Label Declare a participant in the sequence with optional alias
actor actor Name Declare a human actor with person icon
solid arrow message A ->> B: Message Send a synchronous message with solid arrowhead
solid dashed message A -->> B: Message Send an asynchronous message with dashed solid arrow
open arrow message A -> B: Message Send a message with open arrowhead (no fill)
open dashed message A --> B: Message Send a dashed message with open arrowhead
cross message A ->x B: Message Send a message with cross/X mark on target
cross dashed message A --x B: Message Send a dashed message with cross on target
open circle message A ->)B: Message Send a message with open circle on target
bidirectional message A <<->> B: Message Send arrows in both directions simultaneously
activate activate Participant Start showing an activation box on a participant lifeline
deactivate deactivate Participant Stop showing the activation box for a participant
autonumber autonumber Add automatic sequential numbers to all messages
autonumber restart autonumber start-value Restart autonumbering from a specific value
loop loop Condition ... end Mark a sequence of messages as repeating in a loop box
alt / else alt Condition1 ... else Condition2 ... end Mark alternative conditional branches (if/else)
opt opt Condition ... end Mark an optional block that may or may not execute
par / and par Task1 ... and Task2 ... end Mark parallel execution of multiple tasks
critical / option critical Requirement ... option Fallback ... end Mark critical section with optional fallback
break break Condition ... end Mark a break point that exits the sequence
rect highlight rect rgb(200, 150, 255) ... end Highlight a region with a colored rectangle
note left Note left of Participant: Text Add a note on the left side of a participant
note right Note right of Participant: Text Add a note on the right side of a participant
note over Note over Participant1,Participant2: Text Add a note spanning over one or more participants
title title Diagram Title Set a title for the sequence diagram
create create Participant Create a new participant during sequence
destroy destroy Participant Destroy/remove a participant from sequence
box grouping box rgb(200, 150, 255) participant A participant B end Group participants in a colored box
