Most useful Mermaid Class patterns

  • class: Declare a class with optional attributes and methods
  • interface: Declare an interface (implemented using stereotype)
  • abstract class: Declare an abstract class using stereotype
  • enum: Declare an enumeration class
  • public visibility: Public member with plus prefix
  • private visibility: Private member with minus prefix
  • protected visibility: Protected member with hash prefix
  • package visibility: Package-private member with tilde prefix

Useful Mermaid Class syntax

Element Syntax What it does
class class ClassName {members} Declare a class with optional attributes and methods
interface class ClassName { <<interface>> } Declare an interface (implemented using stereotype)
abstract class class ClassName { <<abstract>> } Declare an abstract class using stereotype
enum class EnumName { <<enumeration>> } Declare an enumeration class
public visibility +memberName: Type Public member with plus prefix
private visibility -memberName: Type Private member with minus prefix
protected visibility #memberName: Type Protected member with hash prefix
package visibility ~memberName: Type Package-private member with tilde prefix
static member $memberName: Type Static/class member with dollar prefix
inheritance Child <|-- Parent Inheritance relationship (is-a)
composition Whole *-- Part Composition relationship (owns)
aggregation Container o-- Item Aggregation relationship (has-a, can exist independently)
association Class1 --> Class2 Association relationship (uses)
dependency Class1 ..> Class2 Dependency relationship (uses temporarily)
realization Class ..|> Interface Realization relationship (implements interface)
multiplicity A "1" --> "0..1" B Add cardinality/multiplicity labels to relationships
relationship label A --> B : label Add descriptive label to relationship
generic class class Name~Type~ Generic/template class with type parameter
namespace namespace Name { class ... } Group classes into a namespace container
annotation class Name { <<annotation>> } Mark class as annotation using stereotype
note Note right of ClassName : text Attach a note comment to a class
click click ClassName "url" "tooltip" Make a class clickable with URL
