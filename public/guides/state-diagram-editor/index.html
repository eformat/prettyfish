Most useful Mermaid State patterns

  • stateDiagram-v2 declaration: Declare a state diagram (v2 is the recommended version)
  • simple state: Declare a simple named state
  • state with label: State with custom display label different from ID
  • start state: Initial state marker
  • end state: Final/exit state marker
  • transition: Transition from one state to another
  • transition with event: Transition with event/trigger label
  • composite state: Nested states inside a composite state

Useful Mermaid State syntax

Element Syntax What it does
stateDiagram-v2 declaration stateDiagram-v2 Declare a state diagram (v2 is the recommended version)
simple state StateName Declare a simple named state
state with label state "Display Label" as stateId State with custom display label different from ID
start state [*] Initial state marker
end state [*] Final/exit state marker
transition StateA --> StateB Transition from one state to another
transition with event StateA --> StateB : event Transition with event/trigger label
composite state state Name { ... } Nested states inside a composite state
fork state fork <<fork>> Parallel fork point for concurrent execution
join state join <<join>> Synchronization point joining parallel branches
choice state choice <<choice>> Decision point for conditional transitions
direction direction LR|TB Set diagram direction (LR=left-right, TB=top-bottom)
note left of state note left of StateName text end note Add note on the left side of a state
note right of state note right of StateName text end note Add note on the right side of a state
hide empty description hide empty description Hide the description area for states without text
state with type state Name <<type>> Add stereotype/type to a state
